★ 요구사항 파악
C  학생 추가
R  학생 전체 목록 출력
    학생 1명 검색
    1등 출력
    평균 출력
    이름으로 검색
U  점수 변경
D  학생 정보 삭제
CRUD == 비즈니스 메서드
	,핵심 관심
	, 비즈니스 로직

자료구조를 선택할지가 중요!
배열[ ]
"배열리스트", 연결리스트, 집합, "맵", 트리, ... (웹개발에서는 " " 표시한 것들이 만힝 사용됨)
	(코테 준비중이시면 5가지 공부)
▶ 컬렉션 프레임워크
	== 틀, 구조, ...
	== 역사, 내용, 코드 : xxx
	== 사용법, 활용처 : O
	▷ 저와 함께하는 예제 위조로 학습을 우선 추천


컬렉션
1) 모든 컬렉션이 toString() 오버라이딩 되어있어서 전체출력이 예쁨
2) 다양한 메서드들이 이미 만들어져 있다.
3) 원래 자료형 제한이 없기에,
		자료형을 "강제"해서 사용함!
		== 제네릭 <타입>

배열 리스트
배열리스트는 배열과 유사하지만
  개수가 무제한 vs 개수가 정해짐
  공통점 : 데이터 관련성 o, 자료형 정해짐
인덱스 컨트롤로 관리

객체는 데이터가 없으면 반환 할 수 없지만 (null)
배열은 항상 데이터가 있다. (0개일 뿐)
	size()로 체크할 것


맵
key(키 값)와 value(데이터 값)으로 이루어진 컬렉션 프레임워크이다.
	키 : 다른 데이터와 중복되지 않음 == PK
	데이터 : 실제 데이터
생성 방식
Map<키 값의 자료형, 데이터 값의 자료형>

맵은 종류가 많아서 new로 만들 수 없다.
	맵 종류에서 하나를 지정하여 만들어야 함
	ex) 포켓몬스터 = new 피카츄();
키 컨트롤러 관리
키와 데이터가 하나의 데이터로 취급된다.

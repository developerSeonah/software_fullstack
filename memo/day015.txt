	== DAO가 두개라는 뜻

BoardDTO
  번호(PK)  제목  내용  조회수  작성자
MemberDTO
  아이디(PK)  비밀번호  이름

요구사항(프로그램이 제공하는 기능)
회원가입  	MemberDAO  C
로그인		MemberDAO  R selectOne
로그아웃		Controller
글작성		BoardDAO  C	
글삭제		BoardDAO  D
전체 출력		BoardDAO  R
글검색 : 제목으로 검색,		BoardDAO  R  selectAll
	작성자이름으로 검색	BoardDAO  R  selectAll
글 1개 선택해서 보기		BoardDAO  R  selectOne
글 내용 변경		BoardDAO  R  selectOne  U
회원탈퇴			MemberDAO  D


model이 많을 때 나누는법
1) 기능별로 나누기 (DAO / DTO)
2) 개념별로 나누기(Board / Member)

view가 많다
 ==> 분업
3가지 방법
1. 팀 이름을 파일명으로
2. UF(user flow)  사용자 입장에서 나누기
3. LP(logic proccess)  개발자 입장에서 나누기

DAO와 controller은 생성자가 들어간다
 DAO는 DTO를 타입으로 하는 배열 리스트 생성과 필요할 경우의 샘플
 controller은 DAO와 View의 모든 클래스를 멤버변수로 받고, 전부 초기화한다.
